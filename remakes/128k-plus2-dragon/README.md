# Saga Dark — +2 128K — Dragon Phase

**▶ Play locally:** `tools/play.sh` from the repo root opens the JSSpeccy 3 embed in your browser (loads the Dragon by default). See [the root README](../../README.md#play-locally) for details.

The **Dragon Phase (Dark 3)** reworked as a standalone +2 128K minigame.
Adds an AY soundtrack (Mountain King by Grieg, public domain, with a
proximity-driven dynamic tempo), a death sting on game-over, and a
victory screen with a dedicated PT3 fanfare on the win path.

## Music & sound

- **Mountain King** (Edvard Grieg, *In the Hall of the Mountain King*,
  Op. 23 No. 7, 1875 — **public domain by age**, Grieg †1907). Composed
  to a PT3 module from scratch by [`tools/pt3-compose-mountain-king.py`](../../tools/pt3-compose-mountain-king.py).
  596 B, 6 patterns of chromatic-modulating motif statements, 3-channel
  AY (bass + lead + arpeggio harmony).
- **Dynamic tempo**: `music_tick` reads `D_DRAC` (`0xA0F0`, the dragon's
  on-screen distance) every frame and writes `pt3.Delay` accordingly —
  ≥30 → calm (delay=5), 20-29 → tense (3), 10-19 → frenetic (2),
  <10 → panic (1). The music speeds up as the dragon closes in.
- **Death sting** (`death_handler`): chromatic Bb4→A4→Ab4→G4 sad-trombone
  on AY channel A. Driven by a state machine inside `music_tick` so it
  runs in IM 2 frames without blocking the main loop.
- **Victory screen** (`victory_handler`): paints the title image, clears
  the bottom third to a black panel, typewriter-prints "FELICIDADES! /
  HAS ESCAPADO DEL DRAGON", swaps the player onto `victory.pt3` (a
  458 B A-major fanfare + Grieg-motif-in-major + cadence, generated by
  [`tools/pt3-compose-victory.py`](../../tools/pt3-compose-victory.py))
  which loops until the user presses the title key, then restores
  Mountain King and returns to the title.
- **PT3 player**: Vortex Tracker II PT3 player r.7 by **Sergey Bulba**
  (2004-2007), public domain. Wrapped in `MODULE pt3` to avoid label
  collisions with the disassembled Dark 3 source.

The original Dark 3 win path printed "PULSA PLAY" and called the
cassette LOAD-BYTES routine to load the next game part. That is
meaningless on +2 128K, so the patched ENDED routine now jumps to
`victory_handler` instead.

## Architecture

- `main.asm` (root) is the **+2 128K wrapper**: bank/slot mapping,
  bootstrap at `0x5B00` (DI / border 0 / SP=`0x813D` / 48K-ROM + bank 0
  paged in / `music_init` / IM 2 install / `JP ENTRY=0x9C40`),
  PT3 player + module + victory module placed in bank 3, `SAVESNA`.
- `src/` is the **divergent fork of the Dark 3 source**:
  - `dragon.asm` — Dark 3 code + data + sprites (25 536 B at `ORG 0x9C40`,
    spans bank 2 + bank 0 in the +2 layout). The original ENDED routine
    is patched at two sites (0x9C48 + 0x9C4B) to route the death and
    win paths to our handlers in `master_im2`.
  - `dark3.asm` — small companion include.
  - `music/master_im2.asm` — IM 2 vector + handler, `music_tick`,
    `music_init` (Mountain King) / `music_init_victory` / `mute_pt3` /
    `music_pause` / `music_resume`, death sting state machine,
    victory handler. Lives in the bank 5 zero gap (0x7777-0x7AFF).
  - `music/mountain-king.pt3` — main soundtrack (596 B, generated).
  - `music/victory.pt3` — victory soundtrack (458 B, generated).
  - `music/pt3_play.asm` — Bulba PT3 player (`ORG 0xC000`, bank 3).
  - **Pre-loaded RAM** at `0x4000-0x9C3F` (23 616 B), split into:
    - `screen.bin` (`0x4000-0x5AFF`, 6 912 B) — screen pixels + attrs.
    - `sysvars-data.asm` (`0x5C00-0x813F`, 9 536 B) — system vars + Dark 3
      boot tables as editable `DEFS`+`DEFB`. Byte-perfect against the
      `dark3.sna` RAM at this range. **Reserves the bank 5 zero gap
      at `0x7777-0x7AFF`** for `master_im2`.
    - `offscreen-title.bin` (`0x8140-0x9C3F`, 6 912 B) — offscreen "TEO,
      THE DRAGON" title image that `INICIO` LDIRs to `0x4000`.
      **⚠️ DO NOT RECLAIM** — the gameplay engine reuses this region
      as a sprite/tile source for sky / ground / dragon-body rendering
      (computed `IX+disp` and table lookups, not literal hex operands,
      so static grep misses them). A ZX0 B+ on-demand pass was attempted
      on 2026-05-12 and reverted after observing the corruption. See
      [`MEMORY-MAP.md`](MEMORY-MAP.md) for the full memory layout.

The Dragon Phase has no in-game text to translate; "TEO, THE DRAGON" on
the title screen is already in English. No language overlay is needed
for this remake — unlike the Film.

## Build

```bash
./build.sh   # produces build/saga-dragon-128k.sna
```

## Run

```bash
../../tools/run-emulator.sh build/saga-dragon-128k.sna
```

## Regenerating the music modules

```bash
# Mountain King (main soundtrack)
python3 ../../tools/pt3-compose-mountain-king.py
# Victory fanfare (win screen)
python3 ../../tools/pt3-compose-victory.py
```

Both scripts emit a PT3-compatible byte stream loadable by Vortex
Tracker II / Pro Tracker 3.5+ and by Bulba's player. After regenerating,
re-run `./build.sh` to bundle the new modules.

## Regenerating the pre-loaded RAM split

If the 48K reference `.sna` ever changes:

```bash
python3 ../../tools/dragon-split-preload.py \
        ../../original-48k/snapshots/dark3/dark3.sna \
        src/
```

The script writes `src/screen.bin`, `src/offscreen-title.bin` and
`src/sysvars-data.asm` and verifies the regenerated layout assembles
byte-perfect against the source `.sna`.

## Regenerating the memory map

After any layout change (new `INCBIN`, moved `ORG`, added buffers):

```bash
./build.sh                                # rebuilds + emits dragon.sym
python3 ../../tools/dragon-memory-map.py --update
```

This refreshes the auto-section of [`MEMORY-MAP.md`](MEMORY-MAP.md).
